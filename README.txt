METHODSPY - LET METHODS SPEAK
=============================

"如果系统是战场，MethodSpy 就是你的战地望远镜。"

MethodSpy 是一个注解驱动的 Java 方法观测小工具。
它让你在不改业务调用姿势的前提下，看到方法执行的关键真相：
- 谁被调用了
- 传进来的参数是什么
- 返回结果是什么
- 花了多少时间

并且，敏感字段自动脱敏，日志格式统一整齐，适合本地调试、联调排查和关键链路巡检。


------------------------------------------------------------
一、这是什么：一个“轻量级方法观测器”
------------------------------------------------------------
MethodSpy 不是大而全的 AOP 平台，也不是沉重的监控系统。
它选择了一条更锋利的路线：小、快、可读、可落地。

核心理念：
1) 使用姿势尽量像“零成本”
   - `@Resource` 注入后照常调用
   - 只在目标方法加 `@MethodSpy`
2) 输出内容尽量“有信息密度”
   - 时间、方法、参数、结果、耗时一次看全
3) 工程边界尽量“明确”
   - 当前基于 JDK 动态代理，推荐接口编程


------------------------------------------------------------
二、能力清单：你现在拿到的功能
------------------------------------------------------------
1) 注解触发监听：`@MethodSpy`
2) Spring 自动代理开关：`@EnableMethodSpy`
3) 支持 `@Resource` 注入后直接调用，无需手工 `Spy.on(...)`
4) 参数脱敏：默认识别 password/pwd/secret/token
5) 豪华日志格式：时间 + 方法 + 参数 + 结果 + 耗时
6) 兼容旧 DSL：保留 `Spy.on(...).before(...).after(...).install()`


------------------------------------------------------------
三、30 秒上手（推荐路径）
------------------------------------------------------------
1) 启用 MethodSpy

```java
@Configuration
@ComponentScan(basePackages = "com.zj.demo")
@EnableMethodSpy
public class AppConfig {
}
```

2) 给方法打点

```java
@Service
class UserServiceImpl implements UserService {
    @Override
    @MethodSpy
    public boolean login(String username, String password) {
        return "tom".equals(username) && "123456".equals(password);
    }
}
```

3) 正常注入，正常调用

```java
@Component
public class UseCaseRunner {
    @Resource
    private UserService userService;

    public void run() {
        boolean ok = userService.login("tom", "123456");
        System.out.println("business: login=" + ok);
    }
}
```

------------------------------------------------------------
五、项目结构（开源友好）
------------------------------------------------------------
- `src/main/java/com/zj/spy`
  MethodSpy 核心实现（注解、自动配置、Bean 后处理器、代理逻辑）

- `src/main/java/com/zj/demo`
  Spring 演示样例（配置、service、调用入口）

- `src/test/java/com/zj/spy`
  契约测试（推荐用法、兼容用法、Spring 注入路径）


------------------------------------------------------------
六、运行方式
------------------------------------------------------------
环境要求：
- JDK 17+
- Maven 3.8+

执行方式：
1) 运行 `com.zj.Main`
2) 或执行 `mvn test`


------------------------------------------------------------
七、工程说明与限制
------------------------------------------------------------
1) 当前实现基于 JDK 动态代理，目标服务建议通过接口暴露
2) 默认输出到 stdout，适合开发和调试场景
3) 尚未接入 slf4j/logback（可按团队规范扩展）


------------------------------------------------------------
八、Roadmap（下一阶段）
------------------------------------------------------------
1) 支持类级别 `@MethodSpy`（自动覆盖类内所有 public 方法）
2) 支持异常日志（失败状态 + 异常摘要 + 耗时）
3) 对接 slf4j，支持日志等级与 JSON 输出
4) 研究 CGLIB/ByteBuddy，拓展无接口场景
